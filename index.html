導師評語生成系統（codex優化版）使用說明

本系統協助導師快速整理學生德行評語、特殊優良表現與日常生活表現資料，並可使用本機規則、Google Gemini API 或 ChatGPT／OpenAI API 進行文字潤飾。老師可以先用結構化欄位輸入學生資料，再產生基礎評語，最後視需要使用 AI 優化成較完整、自然的成績單評語。

線上版網址：
https://froglssh.github.io/homeroom-comment-generator-online/

一、開啟系統

  1. 使用 Chrome、Edge、Safari 等瀏覽器開啟線上版網址。
  2. 建議使用電腦或平板操作，畫面較寬時左右兩欄會比較容易填寫。
  3. 若頁面剛發佈後出現 404，請等待 1 至 3 分鐘後重新整理。

二、基本設定

在左側「基本設定」區填寫學生姓名、性別與語氣風格。語氣風格可選擇「溫暖鼓勵」、「客觀中立」或「委婉規勸」，這些設定會影響基礎評語與 AI 潤飾後的文字語氣。

匯入單一學生 Excel

若老師手上有其他校務系統匯出的單一學生 Excel 檔，可在「基本設定」區使用匯入功能。系統會分析 Excel 內的工作表、欄位標題與儲存格文字，嘗試比對學生姓名、學號、評語代碼、幹部職務、競賽活動、學生自評、特殊表現等資料，並列出可匯入的欄位與位置，讓老師確認或手動調整後再套用。

此功能適合匯入單一學生的完整資料；若是整班 Google 表單回覆或學生自評總表，請使用「學生德行自評資料」區的 Excel 匯入功能。

三、選擇學校評語代碼

「學校評語代碼」區可多選學生適合的德行特質，例如性情、向學、進取、紀律、交友、服務、擅長等類別。點選分類後，再點選代碼項目即可加入；若選錯，可按標籤上的叉號移除。

四、填寫職務與特殊表現

擔任職務

可使用快速選單選取班級幹部或科目小老師，例如班長、副班長、衛生股長、生物小老師等，也可以直接自行輸入其他職務。選好職務後，再選擇表現狀況並按加號加入。

校內外活動／競賽

若學生有科展、學科競賽、社團成果、校內外比賽等表現，可輸入活動名稱與成績。系統會在「競賽與活動」欄位顯示，也會整合到「特殊優良表現」中。

例如：活動名稱「校內科展」、成績「特優」，會整理為「校內科展獲得特優」。

五、匯入學生德行自評資料

  1. 按「選擇檔案」上傳 Excel、XLS 或 CSV。
  2. 系統會讀取第一個工作表。
  3. 若表格中有「姓名」或「學生姓名」欄位，系統會嘗試自動比對目前學生。
  4. 也可以從「選擇學生列」手動選取學生。
  5. 按「套用選取學生資料」後，資料會自動帶入自評欄位。

若表單某一欄沒有填寫，匯入後會保持空白，老師可自行補寫。系統可辨識學校服務、社團表現、學業表現、生活常規、社會服務、活動表演、科學展覽、其他特殊表現等常見欄位。

六、智慧生成選項

「智慧生成選項」提供常見學生表現項目，老師可快速點選「佳」或「待改進」。類別包含學習態度、生活常規與守法、人際互動、個人特質。系統會依照所選內容產生基礎評語素材。

七、導師勉勵／結語

若老師希望指定評語結尾，可在「導師勉勵／結語」欄位自行輸入。若留空，系統會依照語氣風格自動產生鼓勵或期許文字。

八、產生基礎評語

填寫左側資料後，按右側「生成基礎評語」。系統會整合學生姓名與性別、評語代碼、智慧生成選項、學生自評資料、幹部職務、活動競賽與導師勉勵，產生初稿。

九、使用 AI 潤飾評語

系統提供「本機規則合併」、「Google Gemini API」與「ChatGPT／OpenAI API」三種潤飾模式。若不想把學生資料送到雲端，請使用本機規則合併；若希望由外部 AI 協助潤飾，可選擇 Gemini 或 ChatGPT／OpenAI。

本機規則合併

不需要 API Key，所有文字只在瀏覽器內處理，不會送到雲端，適合快速合併基礎評語與補充觀察。

Google Gemini API

  1. 在「AI 設定」中選擇「Google Gemini API」。
  2. 按「取得 API Key」前往 Google AI Studio。
  3. 建立並複製 API Key。
  4. 回到系統，貼到 API Key 欄位。
  5. 可按「儲存 Key」將 API Key 儲存在本機瀏覽器。
  6. 按「AI 智慧潤飾與合併」產生最終德行評語。

ChatGPT／OpenAI API

  1. 在「AI 設定」中選擇「ChatGPT／OpenAI API」。
  2. 按「取得 API Key」前往 OpenAI API Key 頁面。
  3. 建立並複製 API Key。
  4. 回到系統，貼到 OpenAI API Key 欄位。
  5. 可按「儲存 Key」將 API Key 儲存在本機瀏覽器。
  6. 按「AI 智慧潤飾與合併」產生最終德行評語。

模型自動選擇與自訂模型

Gemini 與 ChatGPT／OpenAI 模式都不再固定顯示預設模型名稱。系統會在使用 API Key 時查詢該服務目前可用的模型，並自動選擇適合文字生成的模型，以降低日後模型升級或舊模型停用造成失敗的機率。

若老師需要指定特定模型，可在「自訂模型名稱（選填）」欄位輸入模型名稱。此欄位留空時，系統會自動查詢並選擇模型；此欄位有填寫時，系統會優先使用老師輸入的模型名稱。

若不確定模型名稱，可按「查模型名稱」連到官方模型清單查詢：

  • Gemini 模型清單：https://ai.google.dev/gemini-api/docs/models
  • OpenAI 模型清單：https://platform.openai.com/docs/models
若使用 Gemini 或 ChatGPT／OpenAI API 潤飾，評語內容會送至對應 AI 服務進行生成。若不想送出資料，請使用「本機規則合併」。API Key 只會儲存在本機瀏覽器，不會上傳到本系統伺服器。

十、最終德行評語

「最終德行評語」是正式使用的主要評語欄位。建議先生成基礎評語，再加入補充觀察，接著使用本機規則、Gemini API 或 ChatGPT／OpenAI API 潤飾，最後由老師檢查並手動微調。

十一、日常生活表現總表（附表選用）

按「同步所有資料至表格」後，系統會整理右側表格，包含評語代碼、幹部經歷、競賽與活動、德行評語、特殊優良表現。

德行評語

此欄位會自動帶入「最終德行評語」的文字。

特殊優良表現

此欄位會整理學生具體的特殊表現與成就，避免和最終德行評語重複稱讚。

林獻升特殊優良表現：擔任衛生服務隊，協助環境打掃；生物社長，辦理採集活動；段考經常前三名；校內科展獲得特優；學科能力競賽獲得一等獎。

十二、步驟四：複製與匯出評語資料

複製

可使用「複製基礎評語」、「複製最終德行評語」或「複製整張表格」。

匯出 Excel

按「匯出 Excel」後，系統會下載 Excel 檔。Excel 採直向排列，每一個輸出項目由上到下排列，並將內容欄位設定為自動換行，避免長評語橫向過寬。

項目內容
學生姓名學生姓名
性別男／女
最終德行評語評語全文
特殊優良表現表現摘要

匯出 PDF

按「匯出 PDF」後，瀏覽器會開啟列印視窗。在列印視窗中選擇「另存為 PDF」即可。

十三、常見問題

Gemini 或 OpenAI API Key 輸入後仍無法潤飾？

請確認 API Key 是否完整複製、帳號或專案是否已啟用對應 API、是否有可用額度，並確認網路可連線到 Gemini 或 OpenAI API。若仍失敗，可先改用「本機規則合併」。

系統顯示模型不支援或模型不存在？

可能是原本的模型已停用、帳號權限不支援該模型，或自訂模型名稱輸入錯誤。建議清空「自訂模型名稱」欄位，讓系統自動查詢目前可用模型；若要指定模型，請按「查模型名稱」到官方文件確認最新名稱後再填入。

AI 生成文字太短或不完整？

系統已加入回覆長度檢查與自動重試。若仍覺得不夠完整，可在「補充觀察」加入更多具體資料，再重新潤飾。

特殊優良表現和最終德行評語內容重複？

「特殊優良表現」設計為簡潔摘要，只列具體項目與成就；評價與稱讚主要放在「最終德行評語」。

十四、建議使用流程

  1. 填寫學生姓名、性別與語氣。
  2. 選擇學校評語代碼。
  3. 輸入幹部、小老師、活動與競賽資料。
  4. 匯入或填寫學生德行自評資料。
  5. 點選智慧生成選項。
  6. 生成基礎評語。
  7. 補充導師觀察。
  8. 使用本機規則、Gemini API 或 ChatGPT／OpenAI API 潤飾。
  9. 同步日常生活表現總表。
  10. 檢查德行評語與特殊優良表現。
  11. 匯出 Excel 或 PDF。

十五、資料與隱私提醒

  • 若使用「本機規則合併」，文字只在瀏覽器內處理。
  • 若使用「Google Gemini API」，評語內容會送至 Google Gemini API 進行生成。
  • 若使用「ChatGPT／OpenAI API」，評語內容會送至 OpenAI API 進行生成。
  • 請避免輸入不必要的敏感個資。
  • API Key 請妥善保管，不要公開分享。
